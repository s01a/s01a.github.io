tinyrss+fever+mercury建立属于自己的rss服务

发表于 2019-06-10 | 分类于 rss | 评论数： | 阅读次数： | 阅读次数：
燕过留痕，风过留声

1.前期尝试lnmp+tinyrss+fever+mercury搭建rss服务踩了很多坑，本次搭建rss服务基于阿里云vps，在Centos7上利用Docker镜像进行安装。
2.参考HenryQW大佬Awesome TTRSS部署教程：https://ttrss.henry.wang/

前言

不知从什么时候开始互联网上就一直在唱衰RSS，但其一直以高效、清爽的阅读取胜，利用碎片化的时间可以在任何时间阅读你的订阅。

步骤

1.安装docker镜像

  • 为了避免安装docker时报错，请先更新一下

    sudo yum update

    • 为了后续安装方便，请先切换成root用户再进行安装

      yum -y install docker

2.通过docker-compose进行部署

  • 下载docker-compose.yml文件到任意目录

  • docker-compose.yml包含4个镜像

  1. TTRSS
  2. PostgreSQL
  3. Mercury Parser API
  4. OpenCC API
  • 修改docker-compose.yml配置文件，务必更改postgres的用户名密码

  • 在部署之前，先安装docker-compose

    pip install docker-compose

  • 通过在终端下执行 docker-compose up -d 完成部署

  • 部署完成后需要打开181端口，由于用的是阿里云vps，进入控制台-实列-添加安全规则

  • 默认通过 your-domain:181 端口访问 TTRSS，默认账户：admin 密码：password，请第一时间更改。

  • wangqiru/mercury-parser-api 及 wangqiru/opencc-api-server 为支持高级功能而加入的可选服务类容器，删除不会影响 TTRSS 基础功能。

3.插件

Mercury全文抓取

全文内容抓取插件，配合单独的 Mercury Parser API 服务器使用。docker-compose 中已经包含了 HenryQW/mercury-parser-api 服务器。

设置步骤

1.在Preferences->Plugins中，勾选启用mercury_fulltext插件

2.在Preferences->feeds->mercury插件中，设置mercury API。这里直接填入”service.mercury:3000”

3.在Preferences->feeds订阅源中，编辑订阅源启用mercury插件

fever多终端同步
设置步骤

1.在Preferences->Plugins中勾选启用fever

2.在Preferences->Fever Emulation中设置密码

3.在Preferences中勾选并启用Enable API

4.多终端同步，以reeder为列。在reeder->fever Sever输入http://your-domain/plugins/fever/，在Email输入登录用户名，在Password输入插件里设置的密码
